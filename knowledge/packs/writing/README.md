# 论文结构、章节呼应与表达边界 / Paper Structure, Section Logic, and Writing Boundaries

## Summary / 简介
This pack focuses on the narrative closure of a paper: how the question is introduced, theory is carried, methods are justified, and results are interpreted.

这个知识包关注单篇论文的叙述闭环：问题怎样被引出、理论怎样被承接、方法怎样被解释、结果怎样被讨论。

## Research Situations / 适用情境
- 材料已经不少，但论文总写不成完整故事
- 知道每一章大概要写什么，却不知道章节之间怎么接
- results 和 discussion 常常混在一起

## Core Concepts / 核心概念
- 八大模块 / eight core modules
- 章节接口 / section interfaces
- 结果与讨论边界 / results-discussion boundary
- 写作顺序 / drafting sequence

## Frameworks / 方法框架
1. **Eight-Module Skeleton**: Use a stable paper skeleton so each section contributes to one argument rather than becoming a standalone memo.
1. **Section Handoffs**: Each section should answer what the next section now needs to justify.

## Expanded Workflow / 扩展工作流
1. 先用八大模块搭一个整体骨架。
2. 给每一节定义唯一任务，而不是把所有内容都塞进来。
3. 检查上一节是否真的为下一节提供了写作前提。
4. 把方法、结果、讨论的边界提前写成规则。
5. 最后再做例句、润色和表达层面的优化。

## Decision Rules / 判断规则
- 先建结构，再修句子。
- 每节必须能回答‘这节为什么存在’。
- 如果结果段已经开始解释意义，说明 discussion 提前了。

## Checklists / 检查清单
- What job does this section do in the overall argument?
- Does the next section logically require what I have just written?
- Am I reporting results or interpreting them?

## Common Pitfalls / 常见误区
- 一上来就追求漂亮表达
- 每节都像独立笔记，缺少论文主线
- 把理论、方法和结果写成并列堆叠
- discussion 只重复 results，不做解释推进
