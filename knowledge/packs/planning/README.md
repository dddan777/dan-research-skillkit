# 科研规划、时间安排与长期推进 / Research Planning, Time Allocation, and Long-Term Execution

## Summary / 简介
This pack focuses on sustained progress instead of short bursts of effort, using stages, weekly rhythms, deliverables, and review loops.

这个知识包关注长期推进而不是短期热情，强调阶段任务、每周节奏、里程碑产出和复盘修正。

## When To Use / 何时调用
- Help me plan my PhD workflow
- I am stuck and not making progress
- 科研时间安排怎么做
- 我总推进不下去

## Research Situations / 适用情境
- 知道要做什么，但一周一周推进不稳
- 任务很多，却没有清晰阶段和里程碑
- 经常忙得很累，但回头看产出不成形

## Core Concepts / 核心概念
- 阶段治理 / stage governance
- 里程碑 / milestone
- 四周推进 / four-week rhythm
- 复盘 / review loop

## Frameworks / 方法框架
1. **Stage-Rhythm-Output**: Every long task needs a stage definition, a short rhythm, and a concrete output.
1. **Bottleneck Review Loop**: Diagnose whether the slowdown comes from unclear direction, weak structure, missing data, or low execution bandwidth.

## Expanded Workflow / 扩展工作流
1. 先确定当前阶段和这个阶段唯一最重要的产出。
2. 把阶段目标拆成四周节奏，再拆成本周推进点。
3. 为每一周指定一个必须完成的可见交付物。
4. 每周复盘一次：哪些工作推进了主线，哪些只是制造噪音。
5. 一旦卡住，优先诊断瓶颈类型，而不是盲目加班。

## Decision Rules / 判断规则
- 没有可见产出，就不算真正推进。
- 周计划必须服务阶段目标，而不是把任务塞满。
- 长期停滞通常不是不努力，而是瓶颈识别错误。

## Checklists / 检查清单
- What is the next visible deliverable?
- Does this week advance a stage or only create noise?
- What bottleneck needs diagnosis before more effort?

## Common Pitfalls / 常见误区
- 只做任务列表，不定义阶段产出
- 把日程填满却没有里程碑
- 反复切换任务导致认知损耗
- 卡住时继续机械堆时间
