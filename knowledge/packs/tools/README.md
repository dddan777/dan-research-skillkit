# 学术工具、模板与 AI 辅助边界 / Academic Tools, Templates, and AI Usage Boundaries

## Summary / 简介
This pack treats tools as support systems rather than substitutes for research thinking, with focus on templates, information organization, and AI boundaries.

这个知识包把工具当成支持系统，而不是研究能力的替代品，重点是模板调用、信息组织和 AI 辅助边界。

## When To Use / 何时调用
- What tools should I use for this research workflow
- Help me choose a template or toolchain
- 科研工具怎么搭

## Research Situations / 适用情境
- 工具很多，但不知道如何组合成稳定工作流
- 手头有模板，却不知道什么时候该用、什么时候不该用
- 希望使用 AI 提升效率，但担心边界模糊

## Core Concepts / 核心概念
- 工具流 / toolflow
- 模板流 / template flow
- AI 辅助边界 / AI assistance boundary
- 研究支持系统 / research support system

## Frameworks / 方法框架
1. **Tool-by-Task Routing**: Choose tools according to the research task they support rather than novelty or trend value.
1. **Template Reuse Ladder**: Start with a stable scaffold, then adapt it to the current stage, audience, and output requirement.

## Expanded Workflow / 扩展工作流
1. 先按任务链划分工具，而不是按软件品牌堆清单。
2. 为每一类任务指定一个默认模板或默认工具。
3. 把 AI 放在整理、改写、核对和清单辅助的位置，而不是让它代替判断。
4. 每次替换工具前，先评估切换成本和长期维护成本。
5. 定期清理无效工具，保持工作流稳定。

## Decision Rules / 判断规则
- 工具应该减少摩擦，而不是制造切换负担。
- 模板优先解决起步成本，不能代替研究判断。
- AI 更适合加速结构化工作，不适合替代学术责任。

## Checklists / 检查清单
- Does this tool reduce friction or create more switching cost?
- What part should remain human judgment?
- Which template can shorten setup without freezing thinking?

## Common Pitfalls / 常见误区
- 为了新工具频繁重构工作流
- 把模板当成内容本身
- 让 AI 越过本应由研究者承担的判断
- 工具过多导致碎片化
