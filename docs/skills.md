# Skills Directory

This page is the routing map for the `dan-*` skills. Use it when you want to choose the right research workflow quickly.

本页是 `dan-*` skills 的路由目录。你不知道先调用哪个 skill 时，就从这里开始。

## Core Routing / 总路由

### [`dan-research`](../skills-src/dan-research/SKILL.md)
- Use it when: you cannot tell whether the main bottleneck is stage, task, or output.
- Typical outputs: stage diagnosis note, next-output list, routing decision.
- Next step: usually hands off to `dan-literature`, `dan-question-gap`, or `dan-planning`.
- Pack: [`knowledge/packs/research`](../knowledge/packs/research/README.md)

## Literature And Framing / 文献与问题形成

### [`dan-literature`](../skills-src/dan-literature/SKILL.md)
- Use it when: you need search terms, screening layers, reading depth decisions, or a literature matrix.
- Typical outputs: keyword grid, screening logic, note fields, literature workflow.
- Next step: often moves to `dan-question-gap` or `dan-paper-writing`.
- Pack: [`knowledge/packs/literature`](../knowledge/packs/literature/README.md)

### [`dan-question-gap`](../skills-src/dan-question-gap/SKILL.md)
- Use it when: a topic is still too broad, or a gap claim sounds vague and unconvincing.
- Typical outputs: one-sentence research question, gap note, contribution ladder.
- Next step: usually moves to `dan-theory` or `dan-design`.
- Pack: [`knowledge/packs/question-gap`](../knowledge/packs/question-gap/README.md)

### [`dan-theory`](../skills-src/dan-theory/SKILL.md)
- Use it when: you need concepts, mechanisms, boundaries, or a cleaner theory contribution statement.
- Typical outputs: concept map, mechanism statement, theory paragraph.
- Next step: usually moves to `dan-design` or `dan-paper-writing`.
- Pack: [`knowledge/packs/theory`](../knowledge/packs/theory/README.md)

## Design And Writing / 设计与写作

### [`dan-design`](../skills-src/dan-design/SKILL.md)
- Use it when: you are choosing between methods or checking whether evidence really matches the question.
- Typical outputs: design alignment table, method-fit note, data-evidence table.
- Next step: often moves to `dan-paper-writing` or `dan-method-atlas`.
- Pack: [`knowledge/packs/design`](../knowledge/packs/design/README.md)

### [`dan-paper-writing`](../skills-src/dan-paper-writing/SKILL.md)
- Use it when: you need a paper outline, section interfaces, or help separating results from discussion.
- Typical outputs: outline, section logic note, paragraph skeleton, writing checklist.
- Next step: often moves to `dan-submission`.
- Pack: [`knowledge/packs/writing`](../knowledge/packs/writing/README.md)

### [`dan-review-paper`](../skills-src/dan-review-paper/SKILL.md)
- Use it when: you are writing a review article and need type selection, screening logic, or synthesis structure.
- Typical outputs: review type decision, screening plan, synthesis frame.
- Next step: often moves to `dan-paper-writing` or `dan-submission`.
- Pack: [`knowledge/packs/review-paper`](../knowledge/packs/review-paper/README.md)

## Communication And Execution / 沟通与推进

### [`dan-submission`](../skills-src/dan-submission/SKILL.md)
- Use it when: you need a submission package, reviewer triage, or a response letter structure.
- Typical outputs: submission checklist, reviewer comment matrix, response skeleton.
- Next step: may hand back to `dan-paper-writing` for manuscript changes.
- Pack: [`knowledge/packs/submission`](../knowledge/packs/submission/README.md)

### [`dan-defense`](../skills-src/dan-defense/SKILL.md)
- Use it when: you need talk structure, slide logic, or scenario-specific defense preparation.
- Typical outputs: talk outline, defense checklist, slide sequence.
- Next step: may loop back to `dan-research` or `dan-paper-writing`.
- Pack: [`knowledge/packs/defense`](../knowledge/packs/defense/README.md)

### [`dan-planning`](../skills-src/dan-planning/SKILL.md)
- Use it when: progress is unstable, milestones are fuzzy, or you keep switching tasks without shipping outputs.
- Typical outputs: milestone map, weekly rhythm, execution recovery note.
- Next step: often routes back to the blocked domain skill.
- Pack: [`knowledge/packs/planning`](../knowledge/packs/planning/README.md)

## Cross-Cutting Support / 横向支持

### [`dan-method-atlas`](../skills-src/dan-method-atlas/SKILL.md)
- Use it when: you need cross-method comparison before committing to one approach.
- Typical outputs: method comparison table, evidence map, method-risk note.
- Next step: usually moves to `dan-design`.
- Pack: [`knowledge/packs/methods`](../knowledge/packs/methods/README.md)

### [`dan-academic-tools`](../skills-src/dan-academic-tools/SKILL.md)
- Use it when: you need templates, toolchains, or clearer AI-use boundaries around a workflow.
- Typical outputs: tool stack, template list, support workflow note.
- Next step: routes into the domain skill that owns the research task itself.
- Pack: [`knowledge/packs/tools`](../knowledge/packs/tools/README.md)

### [`dan-upgrade`](../skills-src/dan-upgrade/SKILL.md)
- Use it when: you maintain this toolkit and need to refresh packs, layouts, or release checks.
- Typical outputs: rebuild plan, update checklist, release validation note.
- Next step: rebuilds `skills-src`, `.claude/skills`, `.agents/skills`, and manifests.
- Pack: [`knowledge/packs/upgrade`](../knowledge/packs/upgrade/README.md)
